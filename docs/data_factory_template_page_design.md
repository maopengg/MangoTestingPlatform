# 数据工厂：状态模板页面设计

## 1. 页面定位

状态模板页面负责把一个或多个工厂实体组合成可复用的业务状态。例如：

- 默认订单
- 已支付订单
- 已完成订单
- 待审批报销单
- 部门已审批报销单
- 财务已审批报销单

测试人员可以在该页面调试运行状态模板，一次性生成完整业务上下文，并验证创建与清理逻辑是否正确。

## 2. 页面入口

```text
数据工厂 / 状态模板
路径：/data-factory/template/index
```

建议页面布局：

```text
左侧：模板列表
右侧：模板详情
  基础信息
  字段覆盖
  创建步骤
  输出变量
  调试运行
```

## 3. 列表功能

列表字段：

| 字段 | 说明 |
| --- | --- |
| ID | 模板 ID |
| 模板名称 | 例如已支付订单 |
| 基础实体 | 例如订单 |
| 所属产品 | 关联产品 |
| 步骤数量 | 创建步骤数量 |
| 清理策略 | 默认清理策略 |
| 状态 | 启用/禁用 |
| 更新时间 | 最近修改时间 |

操作：

- 新增模板
- 编辑模板
- 复制模板
- 删除模板
- 启用/禁用
- 预览数据
- 调试运行
- 清理调试数据

筛选：

- 所属项目/产品
- 模板名称
- 基础实体
- 状态

## 4. 基础信息

表单字段：

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| 所属项目产品 | 是 | 关联 `ProjectProduct` |
| 模板名称 | 是 | 例如已支付订单 |
| 基础实体 | 是 | 选择工厂实体 |
| 默认清理策略 | 是 | 执行结束/手动/不清理 |
| 失败策略 | 是 | 失败终止/失败跳过 |
| 描述 | 否 | 业务说明 |
| 状态 | 是 | 启用/禁用 |

校验：

- 同一实体下模板名称唯一。
- 基础实体必须启用。

## 5. 字段覆盖

字段覆盖用于在模板中覆盖基础实体的默认字段规则。

示例：基础实体 `订单` 默认 `status=pending`，模板 `已支付订单` 覆盖为：

```json
{
  "status": "paid"
}
```

页面字段：

| 字段 | 说明 |
| --- | --- |
| 字段名 | 来自基础实体字段 |
| 默认规则 | 工厂实体中的规则 |
| 覆盖方式 | 不覆盖/固定值/表达式/依赖字段 |
| 覆盖值 | 根据覆盖方式展示 |

类型校验与工厂实体页面保持一致。

## 6. 依赖编排

当前版本暂未实现独立的模板步骤表。复杂依赖通过字段规则中的“依赖实体字段”完成。

例如 `默认订单` 模板：

| 当前实体 | 字段 | 生成方式 | 依赖模板 | 取值字段 |
| --- | --- | --- | --- | --- |
| 订单 | `user_id` | 依赖实体字段 | 默认用户 | `id` |
| 订单 | `product_id` | 依赖实体字段 | 默认产品 | `id` |

执行效果：

```text
执行默认订单模板
  ↓
生成 user_id，发现依赖默认用户模板
  ↓
上下文无用户则先创建用户
  ↓
生成 product_id，发现依赖默认产品模板
  ↓
上下文无产品则先创建产品
  ↓
创建订单
```

后续如果需要显式多步骤编排，可新增 `DataFactoryTemplateStep`，但当前实现不包含该模型和页面。

## 7. 输出变量

输出变量决定模板执行后哪些字段会进入执行上下文。

默认输出：

- 主键字段。
- 唯一字段。
- 用户勾选的字段。

页面字段：

| 字段 | 说明 |
| --- | --- |
| 上下文名称 | 默认使用模板名称 |
| 字段 | 例如 order_no |
| 输出名 | 例如 order_no |
| 是否输出 | 是/否 |

使用示例：

```text
${{默认订单.id}}
${{默认订单.order_no}}
${{默认用户.id}}
${{默认产品.price}}
```

## 8. 数据预览

状态模板在调试运行前需要支持“预览数据”，用于在不落库的情况下展示当前模板会生成的 JSON。

预览目标：

- 展示即将写入主表的 `payload`。
- 展示每个字段的生成值。
- 展示测试数据方法 `${{}}` 替换后的真实值。
- 标记必填但生成结果为空的字段。
- 标记依赖字段是否已配置依赖模板。
- 递归展示依赖模板的预览结果。
- 不创建 `DataFactoryExecution`，不 insert 数据，不产生清理记录。

预览流程：

```text
点击预览数据
  ↓
选择测试环境
  ↓
解析实体逻辑数据源对应的真实数据库
  ↓
根据字段规则生成 payload
  ↓
递归预览依赖模板
  ↓
返回 payload、字段明细、缺失字段、依赖预览
```

预览结果示例：

```json
{
  "payload": {
    "order_no": "AUTO_ORDER_NO_1A2B3C4D",
    "product_id": "${{产品.id}}",
    "user_id": "${{用户.id}}",
    "quantity": 3
  },
  "missing_fields": [
    {
      "field": "user_id",
      "message": "依赖字段未配置依赖模板 template_id"
    }
  ]
}
```

页面展示：

| 区域 | 说明 |
| --- | --- |
| 字段明细 | 字段名、生成值、状态、说明 |
| payload JSON | 当前模板最终会生成的数据 |
| missing_fields | 必填缺失或配置错误字段 |
| dependencies | 依赖模板的预览结果 |

## 9. 调试运行

模板详情页需要支持调试运行，方便在接入其他调用方前验证数据创建、变量输出和清理逻辑。

调试流程：

```text
点击调试运行
  ↓
选择测试环境
  ↓
可先点击预览数据确认 payload
  ↓
输入临时字段覆盖
  ↓
根据实体逻辑数据源 + 测试环境解析真实数据库
  ↓
执行创建步骤
  ↓
展示上下文 JSON
  ↓
展示创建数据明细
  ↓
允许一键清理
```

调试结果展示：

| 字段 | 说明 |
| --- | --- |
| 执行编号 | 本次调试 ID |
| 执行状态 | 成功/失败 |
| 上下文 | JSON |
| 创建明细 | 每个实体的主键和唯一值 |
| 实际数据库 | 本次创建真实使用的 Database |
| 清理状态 | 未清理/已清理/失败 |
| 错误信息 | 失败原因 |

## 10. 调用边界

状态模板页面只负责模板自身的配置、调试运行和清理验证。

调用方集成不在本页面设计范围内。状态模板页面只输出可执行模板、调试上下文和清理结果。

## 11. 接口设计

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| GET | `/data-factory/template` | 模板列表 |
| POST | `/data-factory/template` | 新增模板 |
| PUT | `/data-factory/template` | 修改模板 |
| DELETE | `/data-factory/template` | 删除模板 |
| POST | `/data-factory/template/copy` | 复制模板 |
| PUT | `/data-factory/template/status` | 启用/禁用 |
| POST | `/data-factory/template/preview` | 预览生成数据，不落库 |
| POST | `/data-factory/template/debug-run` | 调试运行 |
| POST | `/data-factory/template/debug-cleanup` | 清理调试数据 |

## 12. 验收标准

- 可以创建“默认订单”模板。
- 可以创建“已支付订单”模板并覆盖 `status=paid`。
- 可以通过依赖字段自动创建用户、产品、订单。
- 可以在调试运行前预览生成 JSON。
- 预览时能看到缺失字段和依赖模板配置问题。
- 可以调试运行模板并看到上下文。
- 可以一键清理调试数据。
- 模板输出变量能进入调试运行上下文。
- 模板页面不维护外部调用关系。
