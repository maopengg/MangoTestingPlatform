# 数据工厂：状态模板页面设计

## 1. 页面定位

状态模板页面负责把一个或多个工厂实体组合成可复用的业务状态。例如：

- 默认订单
- 已支付订单
- 已完成订单
- 待审批报销单
- 部门已审批报销单
- 财务已审批报销单

测试人员可以在该页面调试运行状态模板，一次性生成完整业务上下文，并验证创建与清理逻辑是否正确。

## 2. 页面入口

```text
数据工厂 / 状态模板
路径：/data-factory/template/index
```

建议页面布局：

```text
左侧：模板列表
右侧：模板详情
  基础信息
  字段覆盖
  创建步骤
  输出变量
  调试运行
```

## 3. 列表功能

列表字段：

| 字段 | 说明 |
| --- | --- |
| ID | 模板 ID |
| 模板名称 | 例如已支付订单 |
| 模板编码 | 例如 order_paid |
| 基础实体 | 例如订单 |
| 默认别名 | 例如订单 |
| 所属产品 | 关联产品 |
| 步骤数量 | 创建步骤数量 |
| 清理策略 | 默认清理策略 |
| 状态 | 启用/禁用 |
| 更新时间 | 最近修改时间 |

操作：

- 新增模板
- 编辑模板
- 复制模板
- 删除模板
- 启用/禁用
- 调试运行
- 清理调试数据

筛选：

- 所属项目/产品
- 模板名称
- 基础实体
- 状态

## 4. 基础信息

表单字段：

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| 所属项目产品 | 是 | 关联 `ProjectProduct` |
| 模板名称 | 是 | 例如已支付订单 |
| 模板编码 | 是 | 例如 order_paid |
| 基础实体 | 是 | 选择工厂实体 |
| 默认别名 | 是 | 注入上下文的默认名称 |
| 默认清理策略 | 是 | 执行结束/手动/不清理 |
| 失败策略 | 是 | 失败终止/失败跳过 |
| 描述 | 否 | 业务说明 |
| 状态 | 是 | 启用/禁用 |

校验：

- 同一产品下模板编码唯一。
- 基础实体必须启用。
- 默认别名不能为空。

## 5. 字段覆盖

字段覆盖用于在模板中覆盖基础实体的默认字段规则。

示例：基础实体 `订单` 默认 `status=pending`，模板 `已支付订单` 覆盖为：

```json
{
  "status": "paid"
}
```

页面字段：

| 字段 | 说明 |
| --- | --- |
| 字段名 | 来自基础实体字段 |
| 默认规则 | 工厂实体中的规则 |
| 覆盖方式 | 不覆盖/固定值/表达式/依赖字段 |
| 覆盖值 | 根据覆盖方式展示 |

类型校验与工厂实体页面保持一致。

## 6. 创建步骤

创建步骤用于表达多实体组合。

例如 `已支付订单`：

| 顺序 | 动作 | 模板/实体 | 别名 | 依赖策略 | 字段覆盖 |
| --- | --- | --- | --- | --- | --- |
| 1 | 创建 | 默认用户 | 用户 | 优先复用已有 | 无 |
| 2 | 创建 | 默认产品 | 产品 | 优先复用已有 | 无 |
| 3 | 创建 | 默认订单 | 订单 | 自动创建 | `status=paid` |

例如 `财务已审批报销单`：

| 顺序 | 动作 | 模板/实体 | 别名 | 字段覆盖 |
| --- | --- | --- | --- | --- |
| 1 | 创建 | 默认用户 | 用户 | 无 |
| 2 | 创建 | 默认报销单 | 报销 | `user_id=${{用户.id}}` |
| 3 | 创建 | 部门审批 | 部门审批 | `reimbursement_id=${{报销.id}}` |
| 4 | 创建 | 财务审批 | 财务审批 | `reimbursement_id=${{报销.id}}` |

步骤配置字段：

| 字段 | 说明 |
| --- | --- |
| 顺序 | 执行顺序 |
| 引用类型 | 工厂实体 / 状态模板 |
| 引用对象 | 选择实体或模板 |
| 输出别名 | 写入上下文的名称 |
| 依赖策略 | 自动创建/优先复用/必须存在/查询已有 |
| 字段覆盖 | 当前步骤临时覆盖字段 |
| 失败策略 | 失败终止/跳过 |

## 7. 输出变量

输出变量决定模板执行后哪些字段会进入执行上下文。

默认输出：

- 主键字段。
- 唯一字段。
- 用户勾选的字段。

页面字段：

| 字段 | 说明 |
| --- | --- |
| 别名 | 例如订单 |
| 字段 | 例如 order_no |
| 输出名 | 例如 订单.order_no |
| 是否输出 | 是/否 |

使用示例：

```text
${{订单.id}}
${{订单.order_no}}
${{用户.id}}
${{产品.price}}
```

## 8. 调试运行

模板详情页需要支持调试运行，方便在接入其他调用方前验证数据创建、变量输出和清理逻辑。

调试流程：

```text
点击调试运行
  ↓
选择测试对象/数据库环境
  ↓
输入临时字段覆盖
  ↓
执行创建步骤
  ↓
展示上下文 JSON
  ↓
展示创建数据明细
  ↓
允许一键清理
```

调试结果展示：

| 字段 | 说明 |
| --- | --- |
| 执行编号 | 本次调试 ID |
| 执行状态 | 成功/失败 |
| 上下文 | JSON |
| 创建明细 | 每个实体的主键和唯一值 |
| 清理状态 | 未清理/已清理/失败 |
| 错误信息 | 失败原因 |

## 9. 调用边界

状态模板页面只负责模板自身的配置、调试运行和清理验证。

调用方集成不在本页面设计范围内。状态模板页面只输出可执行模板、调试上下文和清理结果。

## 10. 接口设计

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| GET | `/data-factory/template` | 模板列表 |
| POST | `/data-factory/template` | 新增模板 |
| PUT | `/data-factory/template` | 修改模板 |
| DELETE | `/data-factory/template` | 删除模板 |
| POST | `/data-factory/template/copy` | 复制模板 |
| PUT | `/data-factory/template/status` | 启用/禁用 |
| GET | `/data-factory/template/step` | 模板步骤列表 |
| POST | `/data-factory/template/step` | 新增步骤 |
| PUT | `/data-factory/template/step` | 修改步骤 |
| DELETE | `/data-factory/template/step` | 删除步骤 |
| PUT | `/data-factory/template/step/sort` | 步骤排序 |
| POST | `/data-factory/template/debug-run` | 调试运行 |
| POST | `/data-factory/template/debug-cleanup` | 清理调试数据 |

## 11. 验收标准

- 可以创建“默认订单”模板。
- 可以创建“已支付订单”模板并覆盖 `status=paid`。
- 可以配置模板步骤自动创建用户、产品、订单。
- 可以调试运行模板并看到上下文。
- 可以一键清理调试数据。
- 模板输出变量能进入调试运行上下文。
- 模板页面不维护外部调用关系。
